1. Create a profile and select Block storage type.
  2. Click on Resource -> Volume Tab -> Create
  3. Select the Availability Zone, Profile and capacity(GB or TB)
  4. Enter a valid name.
  5. Optionally you can "Configure Replication" by checking the check-box.
  6. If you checked the checkbox then select the profile to use and enter the period for replication.
  7. Click Ok. You should see the Volume listed in the table.
  1. Add a backend from the Home Screen.
  2. Go to Resource -> Bucket tab. Click on Create.
  3. In the dialog window that opens enter a valid name, select the backend type and choose the backend where the bucket should be created. All objects uploaded to this bucket will be saved to this backend by default.
  4. SODA supports server side encryption(SSE) on buckets. To enable SSE change the "Enable Encryption?" switch to yes. Select SSE from the list.
  5. Click on Ok. Hooray! your bucket has been created. Now lets upload some objects!

  1. Add two backends from the Home Screen. One for the source and one for the destination buckets.
  2. Go to Resource -> Bucket tab and click on the "Migrate" link under operation.
  3. Enter a plan name. Use a name that will tell you what the migration does. It will help in the future to run the job again.
  4. The source bucket is automatically selected. Select the destination bucket.
  5. By default the migration is scheduled to run immediately. To schedule the migration at a different date and time, uncheck the "Immediately" checkbox and select the data and time.
  6. Optionally you can decide if you want to retain the object at the source or not. If you want to move the object then select the "Delete source objects..." checkbox.
  7. Click on Ok. Depending on the number of objects and the size of objects, the Migration will be started at scheduled time (or immediately).
  8. You can check the migration details under the Dataflow page. In the table, expand to view the details of the migration task.
  9. You can see the source, the destination, the total size of the objects to be migrated, the size of objects currently migrated and the progress of the migration.
